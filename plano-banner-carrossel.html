Valor & Usabilidade

O Conceito do Banner Destaque

O banner do topo é o ponto de maior tráfego visual na plataforma Focatto. Ele permite que lojas, luthiers premium ou professores de alta conversão aluguem esse espaço de forma rotativa (automática), garantindo visibilidade total aos seus anúncios e ao seu perfil.

Para evitar fadiga visual e manter a estética premium, o banner é exibido como um carrossel que altera de slide a cada 5-7 segundos e suporta até um máximo de 5 a 10 parceiros simultâneos (definidos no painel administrativo).

Benefícios para o Anunciante:

  • Impressão Imediata: Todo usuário que abre a página visualiza a marca.
  • Integração de Catálogo: Mostra produtos e serviços dentro do próprio banner.
  • Link Direto: Encaminha o tráfego para a loja interna do vendedor.
  • Autoridade de Marca: Aumenta a percepção de confiança do profissional.
  • Filtro Localizado: Opção de exibir banners baseados no estado/cidade do visitante.
  • Taxa de Cliques (CTR) Elevada: Tráfego altamente qualificado.
Arquitetura de Software

Como Implementar no Firestore & Next.js

A arquitetura é simples e reaproveita as tecnologias já utilizadas no projeto (Next.js, Tailwind, Firebase Firestore).

Podemos criar uma coleção dedicada /banners no Firestore. Isso otimiza o número de leituras, pois a homepage apenas consulta os documentos dessa coleção que estão ativos.

{
  "userId": "uid_do_usuario_anunciante",
  "title": "Guthier Luteria",
  "subtitle": "Regulagem profissional de instrumentos de corda",
  "coverUrl": "https://storage.firebase.com/...",
  "ctaLink": "/vendedor/uid_do_usuario_anunciante",
  "badgeText": "Luthier Premium",
  "active": true,
  "startDate": Timestamp,
  "endDate": Timestamp,
  "featuredProductIds": ["prod_id1", "prod_id2"],
  "impressions": 1420,
  "clicks": 98
}

No arquivo da Home (src/app/page.tsx), puxamos os banners ativos no useEffect e montamos o carrossel.

import { collection, query, where, getDocs } from "firebase/firestore";
import { db } from "../firebase";

// Busca os banners ativos no Firestore
useEffect(() => {
  async function loadBanners() {
    const q = query(
      collection(db, "banners"), 
      where("active", "==", true)
    );
    const snap = await getDocs(q);
    const list = snap.docs.map(doc => ({id: doc.id, ...doc.data()}));
    setBanners(list);
  }
  loadBanners();
}, []);

As regras de segurança devem permitir a leitura pública dos banners ativos, mas limitar a gravação apenas aos administradores do sistema.

match /banners/{bannerId} {
  allow read: if true;
  allow write: if request.auth != null && 
    get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == "admin";
}

No painel do administrador da Focatto (src/app/admin), cria-se um formulário de controle de banners para:

  • Cadastrar o banner vinculando-o ao UID de um usuário.
  • Definir a data de expiração (fim do período pago).
  • Upload da imagem de capa (salva no Firebase Storage).
  • Visualização de estatísticas básicas (impressões e cliques) registradas via triggers na plataforma.
